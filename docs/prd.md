# 需求文档

## 1. 应用概述

### 1.1 应用名称
车机投屏助手

### 1.2 应用描述
一款专为车机安卓系统设计的多显示器投屏工具，支持检测连接的显示器设备，并将指定App投屏至选定显示器，可自定义投屏分辨率和显示比例。支持画面录制、截图、画质优化、延迟调整、历史记录保存、自动匹配推荐、音频路由控制、画面旋转镜像及远程网络投屏等高级功能。

### 1.3 应用类型
移动端App（Mobile App）- 车机安卓原生应用

## 2. 用户与使用场景

### 2.1 目标用户
车机系统用户，需要将特定应用内容投屏到车载副屏或外接显示器的场景。

### 2.2 核心使用场景
- 将导航App投屏至副驾驶屏幕
- 将视频App投屏至后排娱乐屏
- 将仪表盘信息投屏至HUD显示器
- 录制或截图投屏画面用于分享或记录
- 通过网络将车机画面投屏至远程设备

## 3. 页面结构与功能说明

### 3.1 页面结构

```
车机投屏助手
├── 显示器列表页
├── 投屏设置页
├── 投屏中页面
├── 历史记录页
└── 远程投屏设置页
```

### 3.2 显示器列表页

#### 3.2.1 显示器检测
- 使用DisplayManager API检测当前设备连接的所有显示器
- 自动获取每个显示器的设备信息

#### 3.2.2 显示器信息展示
- 以列表形式展示所有检测到的显示器
- 每个显示器项显示以下信息：
  - 显示器编号（Display ID）
  - 分辨率（宽×高，单位：像素）
  - 物理尺寸（宽×高，单位：英寸）
  - 显示器状态（已连接/未连接）
  - 推荐标识（系统根据当前要投屏的应用类型自动推荐最适合的显示器）

#### 3.2.3 显示器选择
- 用户点击列表中的显示器项，进入投屏设置页

#### 3.2.4 历史记录入口
- 提供「历史记录」按钮，点击进入历史记录页

#### 3.2.5 远程投屏入口
- 提供「远程投屏」按钮，点击进入远程投屏设置页

### 3.3 投屏设置页

#### 3.3.1 目标显示器信息
- 显示用户选择的目标显示器基本信息

#### 3.3.2 投屏参数设置

**分辨率设置**
- 提供分辨率输入框，用户可输入自定义宽度和高度值
- 单位：像素

**显示比例设置**
- 提供比例输入框，用户可输入自定义宽高比例
- 格式示例：16:9、4:3、自定义数值

**画质优化设置**
- 提供画质档位选择：流畅、标清、高清、超清
- 提供编码格式选择：H.264、H.265
- 提供码率调节滑块，范围1Mbps-20Mbps

**延迟调整设置**
- 提供延迟调节滑块，范围0ms-500ms
- 显示当前延迟数值

**旋转与镜像设置**
- 提供旋转角度选择：0度、90度、180度、270度
- 提供镜像开关：水平镜像、垂直镜像

**音频路由设置**
- 提供音频输出选择：主屏扬声器、投屏显示器扬声器、同时输出

#### 3.3.3 应用选择
- 使用ActivityManager API获取当前设备已安装的所有应用列表
- 以列表形式展示应用，每项包含：
  - 应用图标
  - 应用名称
  - 包名
- 用户点击选择要投屏的目标应用

#### 3.3.4 预设方案
- 提供「保存为预设」按钮，用户可将当前所有设置参数保存为预设方案
- 提供预设方案列表，用户可快速加载已保存的预设方案

#### 3.3.5 开始投屏
- 用户完成所有设置后，点击「开始投屏」按钮
- 系统根据设置参数启动投屏功能

### 3.4 投屏中页面

#### 3.4.1 投屏状态显示
- 显示当前投屏的目标显示器信息
- 显示当前投屏的应用名称
- 显示当前投屏分辨率和比例参数
- 显示当前画质档位、编码格式、码率
- 显示当前延迟数值
- 显示当前旋转角度和镜像状态
- 显示当前音频输出路由

#### 3.4.2 投屏控制
- 提供「停止投屏」按钮，用户点击后结束投屏并返回显示器列表页
- 提供「截图」按钮，用户点击后截取当前投屏画面并保存至本地
- 提供「录制」按钮，用户点击后开始录制投屏画面
- 录制过程中按钮变为「停止录制」，点击后结束录制并保存视频文件至本地

### 3.5 历史记录页

#### 3.5.1 历史记录列表
- 以列表形式展示所有历史投屏记录
- 每条记录包含：
  - 投屏时间
  - 目标显示器信息
  - 投屏应用名称
  - 投屏参数（分辨率、比例、画质等）

#### 3.5.2 快速投屏
- 用户点击历史记录项，系统自动加载该记录的所有参数并进入投屏设置页

#### 3.5.3 记录管理
- 提供删除功能，用户可删除单条或全部历史记录

### 3.6 远程投屏设置页

#### 3.6.1 网络投屏模式选择
- 提供「作为发送端」选项：将车机画面投屏至远程设备
- 提供「作为接收端」选项：接收远程设备投屏至车机

#### 3.6.2 发送端设置
- 显示当前设备IP地址和投屏端口
- 提供二维码，供接收端扫码连接
- 提供手动输入接收端IP地址和端口的输入框

#### 3.6.3 接收端设置
- 提供扫码功能，扫描发送端二维码自动连接
- 提供手动输入发送端IP地址和端口的输入框

#### 3.6.4 连接与投屏
- 用户完成设置后，点击「开始连接」按钮
- 连接成功后，根据模式启动远程投屏功能

## 4. 业务规则与逻辑

### 4.1 显示器检测规则
- 应用启动时自动执行显示器检测
- 使用DisplayManager监听显示器连接状态变化，实时更新列表
- 仅显示状态为「已连接」的显示器

### 4.2 显示器自动匹配推荐规则
- 根据用户选择的投屏应用类型，系统自动推荐最适合的显示器
- 推荐逻辑：
  - 视频类应用推荐分辨率最高的显示器
  - 导航类应用推荐物理尺寸最大的显示器
  - 游戏类应用推荐刷新率最高的显示器
- 推荐的显示器在列表中显示推荐标识

### 4.3 投屏实现规则
- 使用Presentation API在目标显示器上创建投屏窗口
- 根据用户设置的分辨率和比例参数调整投屏窗口尺寸
- 使用ActivityManager启动用户选择的目标应用，并将其内容渲染至投屏窗口
- 根据画质优化设置调整编码参数和码率
- 根据延迟调整设置控制投屏缓冲时间
- 根据旋转与镜像设置对投屏画面进行变换处理
- 根据音频路由设置控制音频输出通道

### 4.4 画面录制与截图规则
- 截图功能：捕获当前投屏窗口画面，保存为PNG格式图片
- 录制功能：使用MediaRecorder录制投屏窗口画面，保存为MP4格式视频
- 截图和录制文件默认保存至设备存储的「车机投屏助手」文件夹

### 4.5 历史记录与预设方案规则
- 每次投屏结束后，自动保存本次投屏的所有参数至历史记录
- 历史记录按时间倒序排列，最多保存100条
- 预设方案由用户主动保存，无数量限制
- 预设方案包含显示器ID、应用包名、所有投屏参数

### 4.6 远程投屏规则
- 使用WebRTC或RTSP协议实现网络投屏
- 发送端将投屏画面编码后通过网络传输至接收端
- 接收端解码并显示接收到的画面
- 支持局域网和互联网投屏

### 4.7 参数校验规则
- 分辨率宽度和高度必须为正整数
- 显示比例必须为有效数值格式
- 必须选择目标应用后才能开始投屏
- 码率必须在1-20Mbps范围内
- 延迟必须在0-500ms范围内

### 4.8 投屏状态管理
- 同一时间仅支持一个投屏任务
- 投屏过程中，用户可随时停止投屏
- 停止投屏后，目标应用继续在主屏幕运行

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 未检测到任何显示器 | 显示器列表页显示提示信息「未检测到外接显示器」 |
| 显示器连接中断 | 自动停止投屏，返回显示器列表页，提示「显示器连接已断开」 |
| 用户输入无效分辨率 | 提示「请输入有效的分辨率数值」，禁止开始投屏 |
| 用户输入无效比例 | 提示「请输入有效的显示比例」，禁止开始投屏 |
| 未选择目标应用 | 提示「请选择要投屏的应用」，禁止开始投屏 |
| 目标应用启动失败 | 提示「应用启动失败，请重试」，返回投屏设置页 |
| 投屏过程中目标应用崩溃 | 自动停止投屏，提示「应用已停止运行」，返回显示器列表页 |
| 录制过程中存储空间不足 | 自动停止录制，提示「存储空间不足，录制已停止」，保存已录制部分 |
| 截图保存失败 | 提示「截图保存失败，请检查存储权限」 |
| 远程投屏网络连接失败 | 提示「网络连接失败，请检查网络设置」，返回远程投屏设置页 |
| 远程投屏过程中网络中断 | 自动停止投屏，提示「网络连接已断开」，返回远程投屏设置页 |
| 历史记录数据损坏 | 提示「历史记录数据异常」，清空损坏记录 |
| 预设方案加载失败 | 提示「预设方案加载失败」，保持当前设置不变 |

## 6. 验收标准

1. 用户打开应用，进入显示器列表页，查看到所有已连接显示器及其分辨率和尺寸信息，系统自动推荐适合的显示器
2. 用户选择一个目标显示器，进入投屏设置页
3. 用户设置投屏分辨率为1920×1080，显示比例为16:9，画质档位为高清，编码格式为H.265，延迟为100ms，旋转角度为0度，音频输出为投屏显示器扬声器
4. 用户从应用列表中选择导航App作为投屏目标
5. 用户点击「保存为预设」按钮，将当前设置保存为预设方案
6. 用户点击「开始投屏」按钮，导航App内容成功投屏至目标显示器
7. 用户在投屏中页面点击「截图」按钮，截图成功保存至本地
8. 用户点击「录制」按钮开始录制，录制30秒后点击「停止录制」，视频文件成功保存至本地
9. 用户点击「停止投屏」按钮，投屏结束并返回显示器列表页，本次投屏记录自动保存至历史记录
10. 用户点击「历史记录」按钮，进入历史记录页，查看到刚才的投屏记录
11. 用户点击「远程投屏」按钮，进入远程投屏设置页，选择「作为发送端」，输入接收端IP地址和端口，点击「开始连接」，连接成功后车机画面投屏至远程设备
